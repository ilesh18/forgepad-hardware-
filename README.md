<b>Hey, I’m building HackPad !</b><br><br>

I’m designing and building a custom macro keyboard from scratch.<br>
HackPad is a 9-key programmable keypad powered by RP2040 and KMK firmware.<br>
Everything — schematic, PCB, enclosure, and firmware — is built by me.<br>
This project is about learning real hardware design and firmware, not tutorials.<br><br>

I’m building HackPad to move beyond frontend-only projects.<br>
To understand embedded systems, USB HID, and PCB design properly.<br>
To ship a real physical product that actually works.<br><br>

<b>What HackPad is</b><br>
A compact, programmable macro keypad.<br>
Built for coding shortcuts, workflows, and automation.<br>
Plug & play as a USB keyboard.<br>
Fully reconfigurable using Python (KMK firmware).<br><br>

<b>What it’s built with</b><br>
RP2040 (Seeed Studio XIAO RP2040)<br>
9 mechanical keys (3×3 layout)<br>
3 SK6812 RGB LEDs<br>
Custom PCB<br>
Custom enclosure<br>
CircuitPython + KMK firmware<br><br>

<b>What it can do</b><br>
Send any key or shortcut<br>
Run text macros<br>
Trigger IDE actions<br>
Act like a mini stream deck<br>
Change behavior by editing Python code<br><br>

<b>What I’m working on</b><br>
Finalizing PCB and enclosure<br>
Adding RGB feedback modes<br>
Designing multiple key layers<br>
Polishing firmware<br>
Turning this into a clean portfolio product<br><br>

<b>Why I built this</b><br>
To learn embedded systems properly<br>
To build real hardware, not simulations<br>
To understand firmware + USB HID<br>
To move beyond frontend-only projects<br><br>

<b>What’s next</b><br>
RGB effects and per-key feedback<br>
Multiple key layers<br>
Rotary encoder version (V2)<br>
OLED display version (V3)<br>
Better enclosure design<br><br>
